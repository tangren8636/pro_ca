# 前列腺癌患者随访记录跟踪 App 功能设计方案

**作者**：Manus AI
**日期**：2026年3月8日

## 1. 项目概述

本应用旨在为前列腺癌患者及其家属提供一个专业、便捷的随访管理工具。通过集成关键临床数据记录、趋势分析、智能提醒以及报告生成等功能，本应用致力于帮助患者更好地管理疾病，提高治疗依从性，并为医患沟通提供客观、全面的数据支持。核心目标是优化前列腺癌患者的长期随访流程，提升生活质量。

## 2. 核心功能模块

### 2.1 PSA 数据跟踪与分析

PSA（前列腺特异性抗原）是前列腺癌随访中最重要的生物标志物之一。此模块将提供全面的 PSA 数据管理功能，以支持患者和医生对病情进展的评估。

*   **数据记录**：用户可以手动输入 PSA 数值及其对应的检测日期。系统将支持多种 PSA 单位（如 ng/mL），并可记录检测机构信息。
*   **趋势图表**：应用将自动生成 PSA 随时间变化的折线图，直观展示 PSA 水平的动态变化趋势。图表将支持时间范围选择（如近3个月、6个月、1年或全部），并可进行缩放和交互式查看。
*   **生化复发预警**：根据国际临床指南，对于接受根治性治疗（如根治性前列腺切除术或放疗）的患者，若 PSA 连续两次升高并达到特定阈值（例如，根治术后 PSA > 0.2 ng/mL [1]），系统将在趋势图上标记预警点，并向用户发出提示，建议及时就医。
*   **PSA 倍增时间 (PSADT) 计算**：系统将自动计算 PSA 倍增时间，即 PSA 水平翻倍所需的时间。PSADT 是评估肿瘤侵袭性和疾病进展风险的重要指标 [2]，系统将提供计算结果，辅助患者了解病情动态。

### 2.2 睾酮数据跟踪与分析

睾酮水平在前列腺癌的内分泌治疗（ADT）中具有关键意义。此模块将帮助患者监控睾酮水平，评估治疗效果。

*   **数据记录**：用户可记录睾酮数值及检测日期。系统将支持多种睾酮单位（如 ng/dL 或 nmol/L）。
*   **去势水平监控**：应用将重点监控睾酮水平是否达到“去势水平”（通常定义为 < 50 ng/dL 或 < 1.7 nmol/L [3]）。在图表上将清晰标注去势阈值，帮助患者直观了解治疗目标达成情况。
*   **去势抵抗预警**：结合 PSA 和睾酮数据，若患者睾酮水平已达去势水平，但 PSA 持续升高，系统将发出预警，提示患者可能已进入“去势抵抗性前列腺癌 (CRPC)”阶段。此预警旨在提醒患者及时与医生沟通，调整治疗方案。

### 2.3 影像学（骨扫描/CT/MRI）管理

影像学检查是评估前列腺癌转移情况的重要手段。此模块将协助患者管理影像学报告。

*   **结果记录**：用户可以上传骨扫描、CT 或 MRI 检查报告的图片或 PDF 文件。同时，可手动录入核心结论，例如“未见明显转移”、“骨转移病灶稳定”、“骨转移病灶增加”等，并记录检查日期和机构。
*   **对比随访**：系统将提供一个时间轴视图，方便用户按时间顺序查看和对比历次影像学检查的报告和结论，从而了解病灶的变化趋势。
*   **检查周期提醒**：根据患者的疾病阶段（如转移性激素敏感性前列腺癌 mHSPC 或转移性去势抵抗性前列腺癌 mCRPC），系统将根据临床指南 [4] 自动提醒下一次影像学检查的推荐时间（例如，mHSPC 患者每 6-12 个月进行骨扫描和 CT/MRI，mCRPC 患者每 3-6 个月进行骨扫描）。

### 2.4 打针（内分泌治疗）跟踪

内分泌治疗是前列腺癌的重要治疗方式，定期注射是关键。此模块将帮助患者管理注射计划。

*   **药物管理**：用户可以记录正在使用的去势药物名称（如亮丙瑞林、戈舍瑞林等）及其剂型（如1个月、3个月或6个月剂型）。
*   **智能提醒**：根据上次注射日期和所选剂型，系统将自动计算并提前推送下一次打针的预警提醒，确保患者按时接受治疗。
*   **注射记录日志**：所有历史注射记录将以列表形式保存，包括药物名称、剂型、注射日期和地点等信息，方便患者在复诊时向医生提供完整的治疗依从性记录。

### 2.5 综合报告与导出

此模块旨在将所有随访数据整合，生成易于理解和分享的报告。

*   **随访周报/月报**：系统将自动汇总选定时间段内的 PSA、睾酮波动、影像学检查结果摘要以及内分泌治疗记录，生成结构化的随访报告。
*   **PDF 导出**：支持将生成的随访报告导出为标准 PDF 格式文件。用户可以直接通过微信、邮件等方式将报告分享给主治医生，便于医生全面了解患者的随访情况。

## 3. 进阶功能（可选）

为了进一步提升用户体验和疾病管理效果，可考虑增加以下进阶功能：

*   **副作用记录**：允许患者记录内分泌治疗常见的副作用，如潮热、疲劳、骨质疏松、性功能障碍等，并可记录其严重程度和发生频率。这有助于医生评估治疗的耐受性，并及时进行干预。
*   **生活质量 (QoL) 评估**：集成标准化的生活质量评估量表，如 EPIC-26 或 IPSS 评分量表 [5]，定期引导患者进行自评。通过量化评估，可更全面地了解治疗对患者生活质量的影响。
*   **知识库**：提供关于前列腺癌疾病知识、随访指南、治疗方案、副作用管理等方面的科普文章和常见问题解答，帮助患者及其家属更好地理解疾病和治疗。
*   **医患沟通接口**：预留与医院信息系统或医生端应用的接口，未来可实现数据同步和在线咨询功能。

## 4. 技术考量

*   **隐私保护与数据安全**：医疗数据属于高度敏感信息，应用必须严格遵守数据隐私法规（如 GDPR、HIPAA 等）。所有用户数据需进行加密存储和传输，并实施严格的访问控制。用户身份验证和授权机制是必不可少的。
*   **多端同步与账户管理**：支持多平台（iOS/Android）使用，并实现数据在不同设备间的实时同步。考虑到患者可能需要家属协助管理，应提供便捷的账户共享或授权功能，确保数据安全的前提下实现多方协作。
*   **用户界面与体验 (UI/UX)**：界面设计应简洁直观，操作流程易于理解，尤其要考虑到老年用户的使用习惯。图表展示应清晰易读，提醒功能应及时且不干扰用户。
*   **可扩展性**：系统架构应具备良好的可扩展性，以便未来根据临床进展和用户需求，方便地增加新的数据指标、评估工具或功能模块。
